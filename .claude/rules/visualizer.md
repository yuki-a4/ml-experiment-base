# visualizer/ の運用ルール

実験結果を可視化する**共通ビジュアライザー**の置き場。**実験ごとに HTML を作らない** —— 可視化は常にここのツールで行う。

- 見せ方ごとに `visualizer/<name>/` を切る（1 用途 = 1 ディレクトリ）。既存のディレクトリは変更せず、新しい見せ方は新設して併存させる。
- **何を作るか・データ形式は今決めない**。タスク（表形式 / 画像 / セグメンテーション等）によって最適な見せ方が全く変わるため、実際に必要になったタイミングで決める。
- `file://` で直接開く HTML からのローカルファイル `fetch()` は CORS でブロックされる（実測済み）。ファイル選択やドラッグ&ドロップで読ませるか、簡易サーバ経由で開く前提で作る。
