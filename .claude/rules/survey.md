# survey/ の運用ルール

外部の情報源（上位解法・Discussion・論文・技術記事・公開 notebook など）を調査したら、その記録を必ずここに保存する。**調査は揮発させない** —— 会話の中で読んで終わりにせず、後から誰でも原典と要約に当たれる形で残す。

- **1 情報ソース = 1 サブフォルダ**: `survey/<slug>/` を切る。slug は `<出所>-<識別子>-<内容>` の形で、見ただけで中身がわかる名前にする（例: `kaggle-disc-12345-cv-strategy/`、`arxiv-2401.01234-tabpfn/`、`blog-1st-place-solution-2024/`）。既存フォルダと重複する調査は新設せず追記する。
- **各サブフォルダに NOTE.md を置く**: 以下を記録する。
  - 出典（URL・タイトル・著者・取得日）
  - 要約（何が書いてあるか）
  - このコンペへの適用可能性（試すなら何を・どう検証するか）
  - 記録者（LLM なら自身のモデル名。例: `— Claude Fable 5`）
  - 取得した資料そのもの（HTML / PDF / コード片など）も同じフォルダに置いてよい。ただし**データセット級の大容量ファイルは置かない**。
- **INDEX.md（索引）を必ず更新する**: サブフォルダを作成・更新したら `survey/INDEX.md` の表に 1 行追記する（リンク + 種類 + 一言要約 + 記録日 + 記録者）。索引は薄く保ち、詳細はサブフォルダ側に書く。
- **出典必須**: URL・名称・取得日を必ず残す。原典に当たれない調査記録は資産にならない。
- **KNOWLEDGE.md との住み分け**: survey/ は「原典に近い生の調査記録」、KNOWLEDGE.md は「検証済み知見の索引」。調査から得た、次の実験の判断を変える知見は KNOWLEDGE.md に 1 行 + survey/ へのリンクで反映する。survey/ の内容を KNOWLEDGE.md にコピペしない。
- **exp（種別: 調査）との住み分け**: 調査系の exp が外部情報を参照した場合も、外部ソースの記録自体は survey/ に置き、REPORT.md からリンクする。exp 側には「その調査で何を結論したか」を書く。
