# KNOWLEDGE.md / knowledge/ の運用ルール

KNOWLEDGE.md は毎セッション読む「索引」として薄く保つ。倉庫にしない。

- **索引であってアーカイブではない**: 1 知見 = 1 行の結論 + 出典リンク（`exp0005` / Discussion URL / 論文名）+ 記録者。詳細は書かず、リンク先（REPORT.md 等）に飛ばす。記録者は行末に添える（LLM なら自身のモデル名。例: `— Claude Opus 4.8`）。
- **重複させない**: 詳細は各 `exp/expNNNN/REPORT.md`（凍結済み）にある。ここへコピペしない。
- **載せる基準は「決定を変えるか」**: 次の実験の判断を変える知見だけ。データを見れば分かる事実（行数・列名など）は載せない。
- **刈り込む**: 後の実験で覆った知見は「〜は誤り（exp0012 で否定）」に更新するか削除する。育てるだけでなく削る。
- **出典必須**: 外部由来（Discussion / 論文 / 記事）は URL・名称を必ず併記する。
- **肥大化したら分割**: KNOWLEDGE.md が 1 画面を超え、カテゴリが自然に分かれてきたら、詳細を `knowledge/`（例: `cv.md` / `features.md` / `models.md` / `data.md`）へ移す。移した後、KNOWLEDGE.md は各ファイルへの目次（1 行 + リンク）として保つ（常時ロードするのは目次だけ）。**先回りして空のカテゴリファイルを作らない**（知見が分散して索引性が落ちる）。実際に移すものが出てから作る。
