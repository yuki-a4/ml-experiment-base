# knowledge/

KNOWLEDGE.md が肥大化したとき、詳細をカテゴリ別に切り出す置き場。

## 使い方（分割の手順）

- **通常は空でよい**。知見はまず `KNOWLEDGE.md` に集約する。
- `KNOWLEDGE.md` が 1 画面を超え、カテゴリが自然に分かれてきたら、詳細をここへ移す:
  - 例: `knowledge/cv.md` / `knowledge/features.md` / `knowledge/models.md` / `knowledge/data.md`
  - 移した後、`KNOWLEDGE.md` は各ファイルへの**目次（1 行 + リンク）**として保つ。常時ロードするのは目次だけ。
- **先回りして空のカテゴリファイルを作らない**（知見が分散して索引性が落ちる）。実際に移すものが出てから作る。
